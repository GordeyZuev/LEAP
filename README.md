# Deep Learning for Educational Video Analysis: Benchmarking ASR Systems and Pipeline Optimization

> Платформа автоматической обработки образовательного видеоконтента: транскрипция, структурирование, педагогическое обогащение.

---

## Научная новизна

**Результат.** На корпусе свыше 700 записей (более 900 часов контента) достигнуто сокращение цикла постпродакшена с 30 минут ручного труда на запись до менее 2 минут при автоматической обработке, с ускорением ~22× и стоимостью ~$0.075 за час контента (транскрипция + извлечение тем + `summary` + вопросы для самопроверки). Система поддерживает до 50 параллельных записей и развёрнута в производственной эксплуатации.

**Вклад.** Впервые реализована end-to-end платформа, позволяющая любому учебному заведению или организации, создающей видеоконтент в масштабе, централизованно загружать записи, автоматически обрабатывать их и публиковать в различные источники по нажатию одной кнопки или по настроенной автоматической загрузке по расписанию. Анализ 15 решений (сервисы транскрипции, low-code платформы) показал, что ни одно не обеспечивает полный автоматизированный цикл: первые ограничиваются только речью-в-текст; вторые – лимитами на размер файлов (100–500 МБ), отсутствием выделенных CPU-воркеров для обработки видеопотока (тримминг) и необходимостью ручных настроек. Демонстрируется, что production-ready масштабируемость не требует компромисса между точностью и функциональностью (Conclusion, статья).

**Количественное обоснование.** Бенчмарк восьми managed `ASR` API обосновывает выбор Fireworks `whisper-v3-turbo`. Препроцессинг аудио даёт снижение тарифицируемой длительности на 10–25% при сохранении точности (<0.5% рост `WER`). Video Vocabulary (`prompt`-based, без `fine-tuning`) снижает `CER` по ключевым словам на 15.7% и `WER` на 3.2% для технических лекций. Опрос после внедрения (n=237): средняя оценка 4.58 из 5, 59.9% респондентов поставили максимальный балл.

### Корпус и метрики

Эксперименты проводились на корпусе **свыше 700 записей** (более **900 часов** контента) с разнообразием по дисциплинам, качеству аудио и спикерам. Метрики (время обработки, `speedup`, стоимость `API`) усреднены по всему корпусу. Измерения выполнялись через встроенный `audit log` этапов пайплайна (millisecond precision).

Типовая конфигурация: облачная VM (Ubuntu 24.04, 8 vCPU, 8 GB RAM), Celery: 6 prefork workers (CPU), 28 threads (async I/O).

### Ограничения воспроизводимости

- Корпус лекций – внутренние записи; полный датасет не публикуется по соображениям приватности.
- Стоимость `API` зависит от тарифов провайдеров на момент запуска.
- `RTF` и латентность `ASR` берутся из документации провайдеров, а не из замеров на нашем корпусе.
- Результаты Video Vocabulary получены на подвыборке 20 технических лекций с терминологией от преподавателей.
