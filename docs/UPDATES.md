### 2026-03-04: v0.9.6.3 — Вопросы для самопроверки, экспорт, улучшения

#### 📝 Вопросы для самопроверки
- **DeepSeek** генерирует 3–4 вопроса по транскрипции (опция `questions_count` в конфиге)
- Переменная **{questions}** в шаблонах описания — форматирование через `questions_display` (format, prefix, separator)
- Обратная совместимость: `questions_display.enabled: false` по умолчанию

#### 📤 Экспорт записей
- **POST /recordings/export** — экспорт в JSON, CSV или XLSX
- Фильтры по ID или по параметрам (template_id, status, даты и т.д.)
- Режимы: short (основные поля + URL платформ) или long (полные детали + вопросы)

#### 🔧 Прочие улучшения
- Ограничения длины title/description для YouTube (100 символов) и VK (128 для title)
- Enum **Granularity** в api/shared/enums.py — единая типизация
- Улучшения topic extraction: GRANULARITY_CONFIG, учёт usage metadata, обрезка main_topic
- Удалён auto_segments.txt из cache (transcription_module)
- Batch Test #4, #5 в BATCH_TESTING.md

## 📅 Февраль 2026

### 2026-02-23: v0.9.6.2 — Строгая типизация, подготовка к фронтенду

API приведён к ещё более строгой типизации, закладываем фундамент для веб-интерфейса.

#### 📐 Строгая типизация
- `PUT /recordings/{id}/config` — RecordingConfigUpdateRequest вместо dict
- `POST /templates/from-recording/{id}` — body schema вместо Query params (TemplateFromRecordingRequest)
- `granularity` — Literal["short", "medium", "long"] в topics/subtitles
- `formats` — list[Literal["srt", "vtt"]] в subtitles endpoints

#### 🔗 Готовность к фронтенду
- `OAUTH_FRONTEND_REDIRECT_URL` — URL фронтенда для OAuth callback (по умолчанию http://localhost:8080)
- Все callback redirects берутся из настроек — единая точка конфигурации

#### 🔍 Фильтры записей
- `GET /recordings` — sort_by: created_at, updated_at, start_time, display_name, status
- `GET /recordings` — status_filter: валидация по ProcessingStatus

### 2026-02-16: v0.9.6 — Шаблоны, распознавание, защита от дубликатов

**Что изменилось для пользователя:**

#### 📝 Точность транскрипции и тем
- Дополнительные термины по предмету — транскрайбер лучше распознаёт специфичные слова (ML, программирование и т.д.)
- Гибкость структуры тем — коротко (много пунктов), средне или подробно (меньше, но детальнее)
- Переменная {summary} в описании — можно подставлять краткое содержание в метаданные
- Единый файл с результатами — темы и краткое содержание в одном месте

#### 🔒 Защита от дубликатов
- Система не позволит создать два шаблона, пресета или автоматизации с одинаковым именем
- То же для учётных записей — нельзя добавить дубликат одной и той же платформы/аккаунта
- При попытке дубликата — понятная ошибка (409) вместо сбоя

#### 📊 Отслеживание операций
- Логи в едином формате — проще искать и анализировать
- Запись времени каждого этапа — видно, где тратится время при обработке
- Zoom: автоматическое обновление токена при истечении — меньше сбоев при скачивании

**Миграция при обновлении:** переименовать `topics.json` → `extracted.json` (если есть), применить миграцию БД

### 2026-02-12: v0.9.5 — Multi-Source Video Ingestion & Yandex Disk

**Главное нововведение:** платформа теперь поддерживает добавление видео из любых источников — не только Zoom. Видео можно добавить по ссылке, импортировать плейлист целиком или загрузить с Яндекс Диска.

#### 📥 Добавление видео по ссылке

- Одно видео по URL — система сама определит платформу (YouTube, VK, Rutube и др.)
- Импорт плейлиста — все видео добавляются отдельными записями, повторный импорт не создаёт дубликатов
- Опция запуска сразу после добавления — без дополнительного шага
- Форматы: видео (mp4) или только аудио для транскрибации
- Выбор качества: от 480p до лучшего

#### 💿 Интеграция с Яндекс Диском

- Добавление по публичной ссылке — без предварительного подключения аккаунта
- Синхронизация папки — периодический импорт новых видео из выбранной папки
- Выгрузка на Диск — структура папок по шаблону (курс, дата), автосоздание
- Токен доступа хранится зашифрованным

#### 🏗️ Единый процесс для всех источников

- Один и тот же цикл: загрузка → обработка → выгрузка — для Zoom, YouTube, VK, Rutube, Яндекс Диска
- Система сама определяет тип источника по метаданным

#### 📖 Документация

- [YT_DLP_GUIDE.md](YT_DLP_GUIDE.md) — как добавлять видео по ссылке (YouTube, VK, Rutube)
- [YANDEX_DISK_GUIDE.md](YANDEX_DISK_GUIDE.md) — как подключить и выгружать на Яндекс Диск
