# 🎓 План ВКР

## 📋 Название темы ВКР

**RU:** Интеллектуальная платформа для обработки образовательного видеоконтента

**EN:** Intelligent Platform for Educational Video Content Processing

---

## 📄 Описание работы

В работе будет разработана универсальная платформа для интеллектуальной обработки образовательного видеоконтента, основанная на современных технологиях искусственного интеллекта и машинного обучения. Система автоматизирует полный пайплайн обработки видеолекций: загрузку из различных источников, обработку видео (удаление пауз, нормализация звука), автоматическую транскрибацию с использованием нейросетевых моделей, извлечение ключевых тем и генерацию метаданных через большие языковые модели, а также публикацию на целевые платформы с автоматически сгенерированными описаниями и таймкодами. Архитектура платформы разработана как расширяемая и модульная, что позволяет легко добавлять новые источники данных и направления публикации.

В рамках исследовательской части работы будет проведен сравнительный анализ современных моделей автоматической транскрибации речи (OpenAI Whisper, Fireworks AI, AssemblyAI) применительно к русскоязычному образовательному контенту с оценкой качества, скорости обработки и экономической эффективности. Практическая апробация платформы будет проведена на лекциях магистерской программы «Искусственный Интеллект» НИУ ВШЭ с обработкой не менее 500 часов видеоматериалов. Ожидаемым результатом является работающая open-source система, демонстрирующая сокращение времени обработки видеолекций на 80%+, а также методические рекомендации по выбору оптимальных AI-технологий для образовательных учреждений.

## 🐛 Проблемы при разработке

### **Rate Limiting API провайдеров**

**Проблема:** Превышение лимитов запросов к внешним API (Fireworks) при обработке большого объема видео.

**Решения:**
- ✅ Увеличено количество попыток с 2 до 3
- ✅ Экспоненциальная задержка между попытками (2s → 4s → 8s)
- ✅ Ограничение параллельных транскрибаций (макс. 2 одновременно)
- ✅ Валидация конфигурации через Pydantic

### **Управление токенами Zoom API**

**Проблема:** Race conditions при параллельных запросах к Zoom API, дублирование запросов на получение токенов.

**Решения:**
- ✅ Централизованный `TokenManager` для каждого аккаунта
- ✅ Автоматическое кэширование и обновление токенов
- ✅ Защита от одновременных запросов (потокобезопасность)
- ✅ Автоматические повторные попытки при ошибках


### **Обработка больших видеофайлов**

**Проблема:** Ограничения API на размер файлов, время обработки, таймауты.

**Решения:**
- Разбиение больших файлов на части
- Компрессия аудио перед отправкой
- Асинхронная обработка с очередями

---

### **Управление состоянием записей**

**Проблема:** Сложность отслеживания статусов обработки, откаты при ошибках.

**Решения:**
- Расширенные статусы пайплайна (`TRANSCRIBING`, `UPLOADING`, `UPLOADED`)
- Нормализованные сущности `source_metadata` и `output_targets` (уникальные таргеты на запись)
- Улучшенное логирование и мониторинг

### **Интеграция с платформами**

**Проблема:** Различия в API разных платформ (YouTube, VK), ограничения, квоты.

**Решения:**
- Модульная архитектура для платформ
- Единый интерфейс для всех платформ
- Обработка специфичных ошибок каждой платформы

### **Качество транскрибации**

**Проблема:** Различия в качестве транскрибации между провайдерами, особенно для русского языка.

**Решения:**
- Сравнительный анализ моделей (исследовательская часть)
- Настройка параметров для каждого провайдера
- Post-processing для улучшения качества
