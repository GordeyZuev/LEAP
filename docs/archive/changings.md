# Изменения в статье

## Исходная версия

- **Заголовок:** писали «Benchmarking and Pipeline Optimization», не добавляли слово ASR Systems.
- **Абстракт:** описывали сравнение deep learning моделей и API; называли баланс Whisper-v3-turbo оптимальным; указывали ускорение 21.4× и 5.93 RUB за час; в конце – про развёрнутую систему и пользу для вузов.
- **Вклад:** перечисляли 7 API и оптимальный trade-off; отдельным пунктом шёл user validation (n=237) с акцентом на педагогическую полезность и средний балл.
- **Related Work:** держали два подраздела (сервисы / Whisper в образовании); не включали блок про русскоязычные модели и оптимизированные пайплайны; коротко объясняли выбор API против своего сервера.
- **Раздел про модели:** называли «Deep Learning Model Analysis», давали одну таблицу цен в RUB за час (60 мин); подчёркивали дешевизну Fireworks и архитектуру turbo.
- **Ограничения:** клали подразделом внутрь экспериментов; не выделяли отдельно этику и данные, различие облачный API vs акустическая модель; по акцентам опирались в основном на Rao, не подключали обзор Hinsvark.
- **Опрос:** местами формулировали как доказательство педагогической ценности («validates», «strong evidence»).
- **Приложения:** не добавляли детальный RTF для self-hosted и примеры промптов.

## Что переработали

### Термины и рамка

- Развели понятия модель и managed API (предобработка, сегментация, логика у провайдера); уточнили scope выводов (модель vs облачный продукт).
- Изменили заголовок: добавили ASR Systems.
- Переписали абстракт и вклад: говорим про коммерческие провайдеры / системы, а не про «голые модели».

### Абстракт и вклад

- Перевели цены и сравнения в USD; оставили курс к рублю там, где нужен пересчёт.
- Уточнили статус опроса: не RCT, а post-deployment feedback.

### Related Work

- Добавили в начале абзац про неродную речь и акценты и процитировали Hinsvark (bias).
- Добавили подраздел: русскоязычные системы и оптимизированные Whisper-пайплайны; подключили литературу.
- Расписали, почему в центре API, а self-hosted – вспомогательный сравнительный блок (не основной объект исследования).
- Добавили ссылку на Distil-Whisper и оперлись на публичные материалы Fireworks, не смешивая это с «открытым Whisper» как с одним чёрным ящиком.

### Таблицы и метрики

- Таблица коммерческих API: $/ min, RTF / TTFT, промпты, языки; ссылки на прайсы и документацию по строкам.
- Развели batch RTF и стриминговый TTFT.
- Добавили таблицу self-hosted (ориентиры по RTF /WER, ссылки на источники).

### Эксперименты

- Описали железо: облачная VM как основа замеров, ноутбук – как локальный инструмент.
- Уточнили методику Video Vocabulary: посчитали метрики на случайной выборке из 20 лекций (разные курсы); термины согласовали с преподавателями.
- Зафиксировали в тексте параметры аудио перед API (16 kHz, mono, MP3).
- Дополнили таблицу результатов строкой про spot-check.

### Ограничения и этика

- Вынесли ограничения в отдельный раздел перед заключением
- Добавили пункты: этика и данные; speaker / акценты (перекликается с абзацем про Hinsvark в Related Work).

### Приложения и литература

- Добавили приложение RTF Measurement Methodology (таблица железа / условий по строкам self-hosted) и приложение Illustrative Prompts (примеры промптов).
- Расширили библиографию. Для web-источников развели официальную документацию провайдеров и сторонние обзоры (например бенчмарк TTFT для Nova-3 – публикация TalkFlow, в библиографии отдельно от официального DeepgramDocs).
